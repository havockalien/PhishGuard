# Operations Runbook

## 1) Architecture Snapshot

- Inference API: AWS Lambda + API Gateway
- Dashboard UI: Streamlit container on EC2
- Threat reporting:
  - MongoDB Atlas (event logging)
  - SMTP (email alert)
  - PDF reports (generated by backend)

## 6) Common Issues and Fixes

### A) Streamlit: Scanner API request failed: {"message":"Service Unavailable"}

Cause:
- Deep scan request timed out or upstream returned 503.

Fix:
- Keep scanner fallback enabled (already implemented).
- Increase Lambda memory/timeout (2048 MB, 90s).
- Keep `SCANNER_TIMEOUT_SECONDS=60` in Streamlit container.

### B) MongoDB TLS handshake failed

Cause:
- Atlas network/TLS/auth mismatch.

Fix checklist:
1. Atlas Network Access: allow EC2/Lambda path (temporary `0.0.0.0/0` for testing).
2. Confirm rotated MongoDB user/password.
3. Ensure URI password is URL-encoded if it contains special chars.
4. Confirm Lambda environment contains `MONGODB_URI`.
5. Temporary debug only: `MONGODB_TLS_ALLOW_INVALID=true`.

### C) Email failed

Fix checklist:
1. Verify Lambda env has `SMTP_HOST`, `SMTP_USER`, `SMTP_PASSWORD`, `SMTP_FROM`, `SMTP_TO`.
2. Use Gmail App Password (not account password).
3. Confirm sender and recipient are valid.

### D) PowerShell Docker multiline errors (`-e is not recognized`)

Cause:
- Used `\` line continuation in PowerShell.

Fix:
- Use one-line command, or PowerShell backtick continuation.

- Rotate all exposed secrets immediately (MongoDB, SMTP, API key).
- Restrict EC2 inbound `8501` to trusted IPs.
- Replace `dev-key` with strong key.
- Prefer secrets manager/parameter store for production secrets.
